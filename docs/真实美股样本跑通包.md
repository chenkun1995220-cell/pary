# 真实美股样本跑通包 V1

为了在不影响正式每周流水线的情况下验证真实数据链路，项目新增了一个独立的美股样本跑通包。第一批样本为：

| 股票 | CIK | 公司 |
|---|---:|---|
| AAPL | 320193 | Apple Inc. |
| MSFT | 789019 | Microsoft Corporation |
| GOOGL | 1652044 | Alphabet Inc. |

## 1. 样本配置文件

| 文件 | 用途 |
|---|---|
| `data/samples/us_real_sample_companies.csv` | 真实样本公司清单，包含 ticker、CIK、公司名、行业和审计/风险字段 |
| `data/samples/us_real_sample_quotes.csv` | 行情、市值和 EV 补充表，包含 price、shares_outstanding、net_debt |
| `real_sample_pack.py` | 校验样本包是否完整、CIK 是否重复、报价表股票是否对齐 |
| `scripts/run_us_real_sample.ps1` | 独立跑通 SEC 导入、行情补充、行业映射、中位数、质检、筛选和周报 |

`us_real_sample_quotes.csv` 当前只放了 ticker 和 USD 货币占位。要得到可用的估值结果，需要补齐：

```text
price
shares_outstanding
net_debt
quote_date
quote_source
updated_at
```

其中：

```text
market_cap = price * shares_outstanding
enterprise_value = market_cap + net_debt
```

行情补充层会先按单位自动换算，再计算市值和 EV。换算后的内部口径为：

```text
price = USD/share
shares_outstanding = shares
net_debt = USD
market_cap = USD
enterprise_value = USD
```

允许的单位写法如下：

| 字段 | 允许值 | 说明 |
|---|---|---|
| `price_unit` | `USD/share` | 单股美元价格 |
| `shares_unit` | `shares`、`thousand_shares`、`million_shares` | 股本单位 |
| `debt_unit` | `USD`、`USD_thousand`、`USD_million` | 净债务单位 |

只要填写了 `price`、`shares_outstanding` 或 `net_debt` 中任意一个数值，就必须同时填写：

```text
quote_date
quote_source
updated_at
```

这样可以避免“价格来自今天、股本来自旧年报、净债务来源不明”的混用问题。

## 4. 导入外部行情 CSV

如果行情数据来自券商、网页表格或手工整理文件，可以先把外部 CSV 转成标准行情补充表。

样例输入：

```text
data/imports/us_quote_import_sample.csv
```

样例表头：

```text
Ticker,Close,Shares(M),Net Debt(M),Date,Source
```

运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\import_quote_csv.ps1 -InputPath F:\chatgptssd\project2\data\imports\us_quote_import_sample.csv -Output F:\chatgptssd\project2\data\samples\us_real_sample_quotes_imported.csv
```

字段别名表：

```text
data/config/quote_field_aliases.csv
```

默认假设：

| 字段 | 默认值 |
|---|---|
| `currency` | `USD` |
| `price_unit` | `USD/share` |
| `shares_unit` | `million_shares` |
| `debt_unit` | `USD_million` |

如果你的外部 CSV 不是“百万股 / 百万美元”口径，需要在导入后的标准表里手工改 `shares_unit` 和 `debt_unit`，再运行待补清单检查。

## 5. 自动补齐美股行情数据

当前已经可以自动补齐美股样本所需的关键行情字段：

| 字段 | 自动来源 |
|---|---|
| `price` | 优先 Stooq 延迟行情 CSV；不可用时回退到 Yahoo Finance chart 最近收盘价 |
| `quote_date` | 价格源返回的行情日期 |
| `shares_outstanding` | SEC Company Facts 的 `EntityCommonStockSharesOutstanding` |
| `net_debt` | SEC Company Facts 中债务相关概念合计减现金及等价物 |
| `quote_source` | 价格源名称 + `SEC Company Facts` |

真实联网补齐前需要设置 SEC User-Agent：

```powershell
$env:SEC_USER_AGENT='your-name your-email@example.com'
```

然后运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\auto_fill_us_real_sample_quotes.ps1
```

默认输出会覆盖：

```text
data/samples/us_real_sample_quotes.csv
```

如果只是试跑，建议指定临时输出：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\auto_fill_us_real_sample_quotes.ps1 -Output F:\chatgptssd\project2\outputs\us_real_sample_quotes_auto_test.csv
```

离线 fixture 模式：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\auto_fill_us_real_sample_quotes.ps1 -FixtureDir F:\path\to\sec_fixtures -PriceFixtureDir F:\path\to\price_fixtures -Output F:\chatgptssd\project2\outputs\us_real_sample_quotes_fixture_test.csv
```

价格 fixture 文件名使用股票代码，例如：

```text
AAPL.csv
MSFT.csv
GOOGL.csv
```

注意：Stooq 和 Yahoo Finance 都不是官方交易所直连数据源，适合当前阶段做延迟行情补齐和流程验证。正式投资结论建议后续替换为更稳定、授权明确的数据源。

## 6. 真实联网试跑

SEC Company Facts 不需要 API key，但自动访问需要提供合适的 User-Agent。先设置：

```powershell
$env:SEC_USER_AGENT='your-name your-email@example.com'
```

然后运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_us_real_sample.ps1
```

脚本会先生成 `quote_gaps.csv` 和 `quote_gaps.md`。如果行情补充表仍有待补字段，脚本会停止，不会继续抓 SEC 或生成评分结果。这样可以避免用不完整行情跑出误导性的估值分数。

如果只是临时试跑，建议指定独立输出目录：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_us_real_sample.ps1 -OutputRoot F:\chatgptssd\project2\outputs\us_real_sample_test

输出全部放在：

outputs/us_real_sample
```

主要输出包括：

| 输出 | 说明 |

| `sec_us_stocks.csv` | 从 SEC Company Facts 标准化后的财报字段 |

## 8. 当前边界

1. 样本包只覆盖美股，不代表全市场覆盖；
2. SEC 只提供财报事实数据，不提供实时股价、市值、股本和完整行业估值中位数；
3. 如果 `us_real_sample_quotes.csv` 仍为空，筛选器会生成结果，但估值评分会因为缺少市值/EV 而明显不完整；
4. 样本试跑不会覆盖 `data/config`、`data/raw` 或正式每周输出目录。

## 9. SEC TTM 财务指标增强层 V1

真实样本流水线现在会在 SEC 标准化之后自动运行 `sec_financial_metrics.py`，输出：

```text
outputs/us_real_sample/sec_us_stocks_metrics.csv
```

自动增强字段：

| 字段 | 计算口径 |
|---|---|
| `revenue_ttm` | 最近财年 + 最新累计期 - 上年同期累计期 |
| `net_income_ttm` | 同上 |
| `operating_cash_flow` | 同上 |
| `capex` | TTM 资本开支，并转换为负数 |
| `gross_margin` | TTM 毛利 / TTM 收入 |
| `ebitda` | TTM 经营利润 + TTM 折旧摊销 |
| `current_ratio` | 最新流动资产 / 最新流动负债 |
| `roic` | NOPAT /（权益 + 债务 - 现金） |
| `net_debt_to_ebitda` |（债务 - 现金）/ EBITDA |
| `revenue_cagr_3y` | 最近财年收入相对三年前的 CAGR |
| `net_income_cagr_3y` | 最近财年净利润相对三年前的 CAGR |

期间口径记录在 `metrics_period_basis`：

| 值 | 说明 |
|---|---|
| `ttm` | 收入和净利润都成功使用 TTM 公式 |
| `annual_fallback` | 缺少可比累计期，回退最近财年 |
| `partial` | 核心期间口径不一致或事实不足 |

`metrics_as_of` 记录增强指标所使用事实的最新报告期。缺失事实时字段保持空白，不用零或猜测值代替。

当前仍不能只靠 SEC Company Facts 稳定得到股息率，因为股息率需要分红口径与同期市值/股价配合，后续在行情合并层单独处理。
