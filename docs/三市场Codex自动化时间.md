# 三市场 Codex 自动化时间

截至 2026-07-18，三市场自动化任务按北京时间每周六下午运行：

| 市场 | 自动化 id | 时间 | 状态 | 说明 |
|---|---|---:|---|---|
| 美股 | `automation` | 14:05 | ACTIVE | 运行 S&P 500 美股低估公司每周筛选。 |
| A 股 | `a-300-3` | 14:10 | ACTIVE | 运行沪深 300 每周筛选。 |
| 港股 | `automation-5` | 14:30 | ACTIVE | 运行恒生大型股/中型股每周筛选，并作为三市场周筛收口任务。 |

市场任务完成后，由独立验收任务继续检查当周产物：

| 用途 | 自动化 id | 类型 | 时间 | 状态 | 说明 |
|---|---|---|---:|---|---|
| 三市场周交付验收跟进 | `automation-2` | heartbeat | 15:00 | ACTIVE | 只读取当次产物，核对同日运行、候选数、交付状态和影子验证进度；不得重跑市场抓取或修改正式模型。 |

旧任务保留但不作为生产入口：

| 自动化 id | 状态 | 说明 |
|---|---|---|
| `a-300` | PAUSED | 旧 A 股任务，界面曾不可见，避免重复运行。 |
| `automation-3` | PAUSED | 旧港股任务，界面曾不可见，避免重复运行。 |

当前生产调度以 Codex 自动化任务为准，旧 Windows 计划任务不作为当前入口。

`codex_automation_audit.py` 会统一审计上述 4 项生产任务：三项市场 `cron` 必须满足各自时间、工作目录、模型和提示词约束；`automation-2` 必须为周六 15:00 的 `heartbeat`，绑定有效目标任务线程，并保留只读验收与正式模型保护边界。任一任务缺失或漂移，周度运维检查不得报告为 `ready`。
