# 中期目标与开发治理规范

## 当前口径

中期目标采用“证据、预测与决策成熟化”，周期为 8 周。开发流程不绑定具体模型名称或版本，也不以模型角色分工模拟协作。当前执行主体根据运行环境能力完成任务，并使用统一质量闸门完成风险分级复核。

每次任务收尾必须报告：

- 当前开发内容所属模块；
- 该模块完成度；
- 当前目标总完成度。

完成度以 `outputs/automation/latest_medium_term_goal_review.json` 中的 `goals[].module`、`goals[].completion_percent`、`overall_completion_percent` 和 `current_target_total_completion_percent` 为准。

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\show_development_closeout.ps1 -GoalCode model_governance_handoff
```

该入口只读取最新中期目标看板，不抓取行情、不重新评分、不修改正式模型参数。

## 中期目标

在 8 周内，把项目从“可以自动产出”推进到“可信、可复核、可连续评估”的三市场低估研究系统：

- 连续验证美股、A 股、港股及统一收口，保证当次产物新鲜且候选数一致；
- 累积 1 周和 1 个月预测成熟样本，样本不足时不形成模型优劣结论；
- 提高候选解释能力，压降风险复核项并补齐投资研究条件；
- 关闭已确认不可得的证据任务，保留受限回测边界；
- 保持开发状态干净、可审计，自动刷新提交前复核；
- 建立不依赖模型版本、能适应环境变化的开发治理纪律。

所有自我分析、预测复盘和优化建议先进入影子层。证据、样本和人工复核同时达标前，不得自动修改正式模型参数或正式评分参数。

## 模型无关治理契约

中期目标和开发治理交接产物必须输出：

- `development_execution_profile=capability_adaptive_single_agent`
- `review_policy=risk_based_independent_verification`
- `environment_compatibility_policy=runtime_capability_detection_with_safe_fallback`
- `model_version_pinned=false`
- `upgrade_compatibility_required=true`
- `development_governance_note`

治理契约含义：

1. 不绑定具体模型名称或版本，模型选择由当前运行环境决定。
2. 执行前识别运行环境能力，包括工具、权限、依赖、数据和可用上下文。
3. 能力不足时采用安全降级或明确失败，不复用旧产物掩盖缺口。
4. 模型或环境升级后必须重跑相同质量闸门，不得降低验收标准。
5. 正式模型修改权限始终独立于开发模型能力，默认保持关闭。

## 开发习惯

### 1. 最小可验证单元

- 每次只推进一个边界明确、可回放的子任务。
- 先写失败测试或可验证验收条件，再完成最小实现。
- 同步记录输入、输出、改动文件、验证命令和失败边界。

### 2. 证据留存

每次改动保留证据三件套：

- 运行日志或命令结果；
- 当次输出文件；
- 新增或更新的自动化测试。

不能用旧的 `latest_*` 产物证明本次运行成功。每周交付必须核对日期、候选数和产物时间线。

### 3. 风险分级复核

- 低风险：文档、只读汇总或不改变行为的局部调整，执行针对性测试。
- 中风险：共享规则、报告契约或自动化入口变化，执行相关模块测试和集成检查。
- 高风险：数据口径、候选筛选、估值、预测、正式发布或跨市场逻辑变化，必须提供独立验证证据、完整测试和失败回退策略。

复核重点包括数据新鲜度、候选解释、回测证据、影子层边界、报告一致性和是否存在误导性结论。

### 4. 收敛迭代

- 一个子任务通过验证后再进入下一任务。
- 测试、文档、脚本入口和产物字段必须同步收敛。
- 不把半成品交给周度流程，也不以“后续再补”绕过当前验收条件。

### 5. 失败回退策略

- 改动前说明影响模块、预期行为和回退路径。
- 任一步失败时停止下游收口，不复用旧结论作为本次结果。
- 回退后重新生成依赖产物并重跑提交前复核。

## 提交与发布闸门

固定执行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_pre_submit_review.ps1
```

- `ready`：允许进入后续提交或周度发布判断。
- `needs_attention`：修复输入、契约或验证缺口后重跑。
- `blocked`：停止发布，不得复用旧产物。
- `sample_accumulating`：只作为观察记录，不形成稳定模型优劣结论。

提交前复核会拒绝遗留固定模型协作字段、固定模型版本、缺失风险复核策略或不完整的升级兼容契约。重新生成中期目标、开发治理交接和提交前复核产物后才能恢复 `ready`。

## 三市场统一纪律

美股、A 股和港股采用同一执行顺序：

1. 数据质量与口径校验；
2. 候选结论与理由生成；
3. 风险分级复核；
4. 周度收口与提交前复核。

所有输出用于研究筛选和人工复核，不构成投资建议。
