# 回测成分证据队列归档说明

## 状态

历史成分证据补强队列已于 2026-07-11 关闭。关闭后的权威状态为 `evidence_ceiling_confirmed`，回测模式为 `limited_verified_only`。

`latest_membership_evidence_gaps.json` 中的缺口仍然保留，用于说明回测证据覆盖边界；它们不再是每周待办，不会进入行动队列，也不会驱动官方历史来源请求。

## 新字段口径

- `membership_evidence_unresolved_gap_count`：真实未解决缺口数，仅作审计。
- `membership_evidence_action_required_count=0`：当前没有自动或每周人工补证任务。
- `membership_evidence_action_queue=[]`：关闭状态下不生成队列。
- `source_evidence_next_action`：保存旧回测摘要中的原始建议，不能驱动行动。
- `recommended_action=maintain_limited_backtest`：维持受限回测。

## 保留工具

以下脚本仍保留，以便未来在用户提供新官方历史证据后进行单次人工复核：

- `run_membership_evidence_import_plan.ps1`
- `run_membership_evidence_supplement_queue.ps1`
- `run_membership_evidence_supplement_batch.ps1`
- `run_membership_evidence_source_intake_status.ps1`
- `run_membership_evidence_apply_preview.ps1`
- `run_membership_evidence_apply_confirmation_status.ps1`
- `run_membership_evidence_approved_apply_plan.ps1`

这些脚本不再属于 `run_weekly_reporting_bundle.ps1`。手工运行它们也不会自动修改 `historical_membership.csv` 或正式模型。

## 固定安全边界

- secondary / weak 证据不得升级为 verified。
- 历史缺口不得从报告中删除或伪装成已解决。
- 正式回测样本不得扩大。
- `historical_membership.csv` 不得自动更新。
- 正式模型和正式评分参数不得自动修改。
- 任何重新开启动作都必须通过策略文件版本变更和人工复核。

## 与当前成分股的区别

本归档只关闭历史证据补强。当前成分股交叉校验仍按周运行；当前来源异常、成分缺失或交叉校验失败仍由当前成分股流程处理。
