# 回测成分证据补强队列

## 用途

该文档说明 `backtest_evidence_review.py` 新增的历史成分证据补强队列口径。它只读取现有 `latest_membership_evidence_gaps.json`，不抓取行情，不重新回测，不修改正式模型参数。

## 输出字段

- `membership_evidence_action_required_count`：需要补强成分证据的缺口总数。
- `membership_evidence_action_queue_count`：已经进入本轮补强队列的数量，目前按缺口报告返回的前 50 项排队。
- `membership_evidence_action_unqueued_count`：仍未进入补强队列的数量。
- `membership_evidence_action_queue`：本轮可处理的补强队列。

## 队列项

每个队列项保留股票代码、公司名、影响周数、当前证据等级、建议动作和建议来源。`supplement_official_spglobal_source` 会被归一为 `supplement_official_membership_source`，表示后续应优先补充官方 S&P Global 成分证据。

## 当前边界

该队列只把证据缺口变成可执行清单，不代表证据已经补齐。只要历史成分证据比例未达到中期目标，`formal_model_upgrade_allowed` 仍保持 `false`。

## 导入计划产物

`membership_evidence_import_plan.py` 会读取 `latest_membership_evidence_gaps.json` 和可选的 `data/config/us_sp500_current_membership_sources.csv`，生成以下产物：

- `outputs/automation/latest_membership_evidence_import_plan.json`
- `outputs/automation/latest_membership_evidence_import_plan.csv`
- `outputs/automation/latest_membership_evidence_import_plan.md`

导入计划只判断本地是否已经存在可用的官方当前成分来源，不会联网抓取，也不会直接修改 `historical_membership.csv`。当前来源包不存在时，队列项会标记为 `missing_current_source`；来源域名不是 `spglobal.com` 或其子域名时，会标记为 `invalid_current_source`。

导入计划会同时输出 `ready_to_import_weeks_affected`、`missing_source_weeks_affected` 和 `invalid_source_weeks_affected`，用于衡量当前队列中有多少回放周的弱证据已经具备导入条件、仍缺官方来源或来源无效。该统计只是影响范围评估，不代表已经改写历史成分证据。

该产物已经纳入 `run_weekly_reporting_bundle.ps1` 和提交前复核。提交前复核会检查它是否存在、结构是否正确、日期是否新鲜，但不会因为 `missing_source_count` 大于 0 而阻断提交；缺来源代表后续研究任务，不代表自动化链路失败。

## 当前来源包接入回测

`run_us_point_in_time_backtest.ps1` 现在会检查 `data/config/us_sp500_current_membership_sources.csv`。如果该文件存在且比 `historical_membership.csv` 更新，回测脚本会重建历史成分输入，并把它传给 `backtest_membership_inputs.py --current-source-pack`。

当前来源包字段建议为：

- `ticker`
- `membership_evidence`
- `membership_source_url`
- `source_as_of_date`
- `notes`

人工录入的 `source_as_of_date` 必须使用 `YYYY-MM-DD` 格式，且不能晚于本次 intake 复核日期。格式错误或未来日期会被标记为 `invalid_source_date` / `invalid_future_source_date`，不会写入 `latest_membership_evidence_verified_source_pack.csv`，也不会进入成员证据导入预演。

只有 `membership_evidence=verified` 且 `membership_source_url` 是 `https://spglobal.com` 或 `https://*.spglobal.com` 时，当前成分证据才会保留为 `verified`；否则会自动降级或保持为 `secondary`。这条通道只用于当前仍在指数内的成分股，不替代历史调入调出事件证据包。

## 官方当前来源抓取器

`sp500_current_membership_sources.py` 和 `scripts/run_sp500_current_membership_sources.ps1` 会尝试从官方 S&P Global 页面解析当前成分 ticker，并写入 `data/config/us_sp500_current_membership_sources.csv`。

该抓取器遵守保守边界：

- 只有来源 URL 是官方 S&P Global HTTPS 域名时才允许生成 `verified`。
- 只有在页面内容中实际解析到模板 ticker 时才写入来源包。
- 当官网直连受限时，可以先从官方 S&P Global 页面手动下载或保存当前成分股 CSV / `.xls` 导出，再通过 `scripts/run_sp500_current_membership_sources.ps1 -SourceFile <官方导出文件>` 导入。该文件可以是普通 CSV，也可以是 S&P Global `file.xls` 这类 Excel 兼容 HTML 表格；需要包含 `Symbol`、`Ticker`、`Ticker Symbol`、`Constituent Ticker` 或 `Constituent Symbol` 等 ticker 列，且仍必须搭配官方 S&P Global `SourceUrl` 才能生成 `verified`。
- 周度收口脚本会自动检查固定入口 `inputs/sp500_current_membership/official_constituents.csv`；如果文件存在，会先执行 dry-run 校验，通过后再传给 `run_sp500_current_membership_sources.ps1 -SourceFile` 正式导入。也可以在收口脚本中显式传入 `-Sp500CurrentMembershipSourceFile <官方导出CSV>`。
- 如果官方页面返回 403、网络失败或页面不可解析，会写出只有表头的空来源包，并在 `outputs/automation/latest_sp500_current_membership_sources.md` 中记录 `status: fetch_failed` 和错误原因。
- 抓取失败不会把任何 ticker 升级为 `verified`，也不会改变正式模型或回测输入。
## 官方页面无明细时的处理

S&P Global 官方 S&P 500 页面有时可以正常打开，并显示 `Full Constituents List` 区域，但公开 HTML 里不一定包含完整成分股明细行。此时 `sp500_current_membership_sources.py` 会输出 `status: source_file_required`，`next_action: provide_official_constituents_csv`，并只写出空来源包，不会把任何 ticker 升级为 `verified`。

继续补强时，应从官方 S&P Global 页面下载或保存当前成分股 CSV / `.xls`，再通过 `scripts/run_sp500_current_membership_sources.ps1 -SourceFile <官方导出文件>` 导入。文件至少需要包含 `Symbol`、`Ticker`、`Ticker Symbol`、`Constituent Ticker` 或 `Constituent Symbol` 等 ticker 列。正式导入前可以先运行 `scripts/run_sp500_current_membership_sources.ps1 -SourceFile <官方导出文件> -DryRun` 做只读预验证；该模式会输出 `validation_only: true`、匹配数量、覆盖状态、实际使用的 `source_file_ticker_columns` 和错误原因，不写入 `data/config/us_sp500_current_membership_sources.csv` 或自动化产物。

来源抓取状态会同步写入 `outputs/automation/latest_sp500_current_membership_sources.json`。该 JSON 是提交前复核和中期目标看板的机器可读输入，用于区分 `ready`、`fetch_failed` 和 `source_file_required`，并保留下一步动作；空 CSV 只表示本批没有 verified 来源行，不再承担解释失败原因的职责。

工具还会输出 `outputs/automation/sp500_current_membership_source_intake_template.csv`，用于人工补源核对。该文件字段是 `expected_ticker`、`intake_status`、`required_source_url`、`required_source_columns` 和 `notes`，刻意不使用可解析的 ticker 列名，避免被误当成官方导出文件导入。真正导入时仍必须使用 S&P Global 官方导出的成分股文件；如果把 intake template 传给 `-SourceFile -DryRun`，工具会返回 `source_file_invalid` 并提示缺少可识别的 ticker 列。

工具还会同步生成 `outputs/automation/sp500_current_membership_source_file_request.md`，用于给人工补源提供一页式请求包。该文件列出当前状态、必须字段、至少 400 个 ticker 的验收标准、dry-run 校验命令、正式导入命令和缺失 ticker 样例。它不是官方来源文件，不能作为 `-SourceFile` 导入，也不会把任何历史成分证据升级为 `verified`。

`scripts/check_sp500_current_membership_source_inbox.ps1` 可单独检查固定入口 `inputs/sp500_current_membership/official_constituents.csv`。它只读官方 CSV 投递入口和 intake template，输出 `outputs/automation/latest_sp500_current_membership_source_inbox_status.json` 与 `.md`，用于区分 `missing`、`invalid`、`incomplete` 和 `ready_for_import_preview`，并记录实际使用的 `source_file_ticker_columns`。该检查不会写入当前来源包、不会改写历史成分文件，也不会修改正式模型参数。

当使用 `-SourceFile <官方导出CSV>` 导入后，工具会先读取已有的人工核对表，再输出覆盖验收字段：`intake_coverage_status`、`intake_expected_count`、`intake_matched_count`、`intake_missing_count` 和 `intake_missing_tickers`。`complete` 表示核对表中的 ticker 已全部被官方文件覆盖，`partial` 表示只覆盖一部分，`none` 表示本批核对表没有任何 ticker 被覆盖。随后核对表会被更新为剩余缺口，便于下一轮继续补源。

当覆盖状态为 `partial` 或 `complete` 时，来源状态 JSON 会给出 `recommended_followup: run_membership_evidence_import_plan_then_apply_preview`。此时应重新运行成分证据导入计划和应用预演；导入计划会优先排列 `ready_current_source`，并在同一状态内按 `weeks_affected` 从高到低排序，使影响回放周数最多的补强项优先进入预演。

每周人工处理清单会读取 `outputs/automation/latest_membership_evidence_import_plan.json`。当 `ready_to_import_count` 大于 0 时，它会追加 `run_membership_evidence_apply_preview` 人工事项，提醒先运行 `scripts/run_membership_evidence_apply_preview.ps1` 并核对预演报告；该动作仍然只做预演，不直接改写历史成员文件，也不允许自动升级正式模型参数。

提交前复核也会做跨产物校验：如果导入计划中 `ready_to_import_count` 大于 0，但 `latest_weekly_action_items.json` 没有 `run_membership_evidence_apply_preview`，则标记 `membership_evidence_apply_preview_action_item_missing`，要求先修复周行动清单再收口。

为便于后续看板读取，提交前复核 JSON 同步输出 `membership_evidence_ready_to_import_count`、`membership_evidence_preview_row_count` 和 `membership_evidence_preview_action_item_present`。这三个字段只描述成分证据补强闭环状态，不代表历史成分文件已经被改写，也不代表可以自动升级正式回测模型。

中期目标看板的 `backtest_evidence_quality.next_action` 会优先读取这些证据闭环状态：如果 `ready_to_import_count` 大于 0，则优先提示 `run_membership_evidence_apply_preview`；如果当前成分来源建议 `run_membership_evidence_import_plan_then_apply_preview`，则提示先重跑导入计划和应用预演；如果当前来源建议 `review_current_membership_source_status`，则提示先核对仍缺失的当前成分 ticker 或来源导入覆盖；否则继续提示 `supplement_verified_membership_evidence`。这些动作都只服务于证据补强和人工复核，不自动改写历史成分文件，不自动升级正式模型。

每周人工处理清单也会读取 `outputs/automation/latest_sp500_current_membership_sources.json`。当该文件给出 `recommended_followup: review_current_membership_source_status` 且仍有 `missing_count` 或 `intake_missing_count` 时，清单会追加 `review_current_membership_source_status`，要求核对 `latest_sp500_current_membership_sources.json`、`missing_ticker_review_queue` 和 `sp500_current_membership_source_intake_template.csv` 中缺失的 ticker。当 `recommended_followup: provide_official_constituents_csv` 时，清单会追加 `provide_official_constituents_csv`，并指向 `sp500_current_membership_source_file_request.md`、dry-run 校验命令和正式导入命令。该动作只用于人工核对当前来源缺口或提供官方 CSV，不改写 `historical_membership.csv`，不放宽正式模型升级门槛。

每周人工处理清单还会读取 `outputs/automation/latest_sp500_current_membership_source_inbox_status.json`，把固定入口是否缺文件、入口校验状态、解析 ticker 数、intake 缺口数和下一步动作写入 `provide_official_constituents_csv`。这样人工只看每周待办即可判断是否需要先投递官方 CSV、替换无效文件，还是进入 dry-run 导入。

提交前复核会检查这个闭环：如果当前来源状态要求 `review_current_membership_source_status`，但每周人工处理清单没有对应动作，则标记 `sp500_current_membership_source_action_item_missing`，要求先重跑或修复 `show_weekly_action_items.ps1`。

## S&P 500 官方 CSV 请求包指纹

`outputs/automation/sp500_current_membership_source_file_request.md` 还必须写出当前入口指纹值：`source_file_inbox_size_bytes: 当前值`、`source_file_inbox_sha256: 当前值或 none`、`source_file_inbox_modified_at: 当前值或 none`。提交前复核会拒绝只列字段名、不写当前值的请求包。

## S&P 500 成分证据来源分级

- `verified`：S&P DJI / S&P Global 官方成分文件、官方指数公告、官方方法库或同域名官方 PDF。只有这一类来源可以把当前或历史成分证据升级为 `verified`。
- `cross_check`：IVV、SPY、VOO 等跟踪 S&P 500 的 ETF 官方持仓文件，只用于交叉核对 ticker、权重和异常缺口，不升级历史或当前成分证据。
- `secondary`：Wikipedia、公开镜像、项目缓存和非官方整理表，只用于解阻运行和发现缺口。

当前 secondary fallback 只解决“自动化能继续运行”的问题，不解除 `formal_backtest_upgrade_allowed=false`。当 `membership_evidence_import_plan` 中仍存在 `invalid_current_source` 或 `cross_check` 来源时，必须继续获取 S&P DJI / S&P Global 官方来源后再进入导入预演。

`sp500_verified_source_plan.py` 会读取当前来源状态、固定 inbox 状态、membership evidence import plan 和 backtest evidence review，输出机器可读的 verified 来源补强计划：

- `outputs/automation/latest_sp500_verified_source_plan.json`
- `outputs/automation/latest_sp500_verified_source_plan.md`

该计划把 S&P Global full constituents export 和官方公告列为可升级的 `verified` 路径，把 IVV、SPY、VOO 官方持仓列为 `cross_check_only`，并给出下一步命令。它只生成计划，不抓取网页，不导入来源，不改写历史成分文件。

```powershell

powershell -ExecutionPolicy Bypass -File scripts\run_sp500_current_membership_sources.ps1 -ProjectRoot F:\chatgptssd\project2 -SourceFileInbox inputs\sp500_current_membership\official_constituents.csv
powershell -ExecutionPolicy Bypass -File scripts\run_membership_evidence_import_plan.ps1 -ProjectRoot F:\chatgptssd\project2
powershell -ExecutionPolicy Bypass -File scripts\run_sp500_verified_source_plan.ps1 -ProjectRoot F:\chatgptssd\project2
powershell -ExecutionPolicy Bypass -File scripts\run_membership_evidence_apply_preview.ps1 -ProjectRoot F:\chatgptssd\project2
```
