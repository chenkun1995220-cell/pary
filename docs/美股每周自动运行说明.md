# 美股每周自动运行说明

## 自动流程

每次运行会按顺序完成以下工作：

1. 刷新标普 500 成分股名单，失败时使用最后一次有效缓存。
2. 从 SEC 更新全部成分公司的基础信息。
3. 自动补齐价格、流通股数、净负债和行情日期。
4. 更新 SEC 财务指标并执行低估评分。
5. 输出候选公司列表、评分理由和候选公司深研包。

股票池按成分表实际证券行数导入，不强制截断为 500 行。类别股代码会转换为 SEC 和 Yahoo 可识别的连字符格式。Company Facts 缓存在 `data/cache/sec_companyfacts`，行情、财务导入和指标计算共享同一份数据。

总控脚本：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_us_universe_weekly.ps1 -SecUserAgent "Your Name your-email@example.com"
```

只检查执行计划，不联网、不写入文件：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_us_universe_weekly.ps1 -SecUserAgent "Your Name your-email@example.com" -DryRun
```

## 自动化任务

当前拆分为三个独立 Codex 自动化任务，均按北京时间每周日运行：

| 时间 | 任务 | 当前能力 |
|---|---|---|
| 14:05 | 美股低估公司每周筛选 | 标普500行情、SEC财务、评分、候选报告和深研包 |
| 14:25 | A股沪深300每周筛选 | 名单、行情、财务质量、增长、估值评分与候选报告 |
| 14:45 | 港股大中盘每周筛选 | 名单、行情、财务质量、增长、估值评分与候选报告 |

旧 Windows 计划任务已禁用。三个总控脚本使用不同防重入锁、缓存和输出目录。

A股和港股当前使用 `regional_fundamental_v2` 模型，综合同行业PE/PB折价、ROE、ROIC、毛利率、资产负债率、流动比率、经营现金流和收入利润增长。候选公司要求PE、PB为正、ROE不低于5%、财务数据可用、行业样本不少于5家且总分不低于75分。

V2模型仍未包含审计意见、治理事件、商誉、股权质押、AH股溢价和完整自由现金流，因此候选结果是基本面量化初筛，不替代原始财报核验。

## 每周自我分析摘要

三市场周筛和严格时点回测完成后，可运行统一自我分析入口：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_self_analysis.ps1
```

该脚本读取三市场 `latest_run_summary.md`、三市场 `model_audit.md`、三市场 `data_health_history.csv`、三市场 `quote_gaps.csv`、三市场 `valuation_review_items.csv` 和美股 `latest_backtest_summary.md`，输出 `outputs/automation/latest_self_analysis.md`、`outputs/automation/latest_self_analysis_manifest.json`、`outputs/automation/latest_manual_review_queue.csv`、`outputs/automation/manual_review_queue_history.csv` 和 `outputs/automation/manual_review_repeats.csv`。自我分析摘要只汇总运行状态、数据健康、缺口和下周优化建议；行情覆盖、财务覆盖、刷新状态或数据质量出现异常时，只进入人工复核，不自动修改正式模型参数。manifest 记录本次日期、输出路径、三市场 `markets` 运行快照、三市场 `health` 数据健康快照、人工复核队列数量、历史重复项数量、`review_status` 和 `recommended_next_action`，供后续自动任务或仪表盘读取。

当 A 股或港股行情缺口属于非正 PE/PB 时，`quote_gaps.csv` 会在 `review_category` 中标记估值复核分类，例如 `loss_making_or_negative_pe`、`non_positive_book_value_or_pb` 或 `special_industry_valuation_review`。自我分析摘要会汇总这些分类，帮助区分亏损口径、净资产口径和特殊行业估值口径，仍只作为人工复核线索。

区域筛选器还会输出 `valuation_review_items.csv` 作为估值复核清单，记录被 PE/PB 质量门挡住的公司、复核分类和排除理由。自我分析摘要会从该清单抽取估值复核样例，展示股票、公司、复核分类和 PE/PB 细节，便于每周快速定位需要人工复核的代表性公司。该清单用于后续人工复核、周度对比和自我分析，不进入候选池，也不改变 `regional_fundamental_v2` 的正式评分参数。

自我分析摘要还会生成“人工复核队列”，优先汇总估值口径样例，再补充候选结论缺口和候选风险提示。该队列同时写入 `latest_manual_review_queue.csv`，字段为生成日期 `as_of_date`、优先级序号、市场、复核类型、股票、公司和复核要点；同时按周归档到 `manual_review_queue_history.csv`，同一生成日期重跑时会替换该日期旧记录，避免重复留痕。自我分析会根据历史归档识别“历史重复项”，把本周再次进入复核队列的股票和历史出现日期列入摘要，并写入 `manual_review_repeats.csv`，用于后续人工判断是否需要调整数据补齐、估值口径或复核流程。该队列用于每周人工复核的阅读顺序和历史对比，不代表自动买入、卖出或模型参数调整。

## 12个月目标价与走势跟踪

三市场周流程会在候选筛选后运行 `valuation_trend_v1`：

- PE 公允价权重 50%，PB 公允价权重 30%，美股自由现金流公允价权重 20%；缺失项目按有效权重重新分配。
- 财务质量调整系数限制在 0.85 至 1.10，12个月目标价不超过当前价的 1.60 倍。
- 高、中、低置信度分别使用 20%、25%、30% 安全边际计算建议买入价。
- 走势依据近一年日线、20/60/120 日均线、12个月动量、波动率和52周位置，分类为偏强、温和偏强、中性、偏弱或数据不足。
- `valuation_targets.csv` 保存本期目标价，`forecast_history.csv` 永久追加每周预测，`valuation_report.md` 输出中文分析。
- `latest_investment_summary.md` 会输出“候选风险说明”和“候选结论质量检查”；风险说明基于估值置信度、安全边际、走势、资产负债率、流动性、增长和数据质量自动生成，质量检查逐家公司标记低估依据、风险说明、建议买入价、目标价、跟踪状态和数据质量说明是否完整。两者只用于人工复核，不改变候选排序，也不自动修改模型参数。
- 候选行情覆盖率低于 80% 时估值步骤失败，不更新本次正式摘要，也不得把旧报告当作本次结果。

## 预测跟踪与模型审计

三市场周流程在目标价生成后继续执行预测跟踪和模型审计：

- 使用股票复权收盘价，并分别对照美股标普500、A股沪深300、港股恒生综合指数。
- 每周更新 `tracking_snapshot.csv`；在预测后4、12、26、52周生成正式评价，写入不可覆盖、按唯一键去重的 `forecast_evaluations.csv`。
- 同时评价绝对收益、相对基准超额收益、目标价误差、最大有利/不利波动和方向命中率；方向变化超过正负5%才判定为上涨或下跌。
- `performance_report.md` 汇总本周跟踪结果，`model_audit.md` 记录每周自我分析，`shadow_model_proposals.csv` 只保存影子分析候选。
- 成熟评价少于30条时只显示“样本积累中”；验证集少于15条或市场覆盖不足2个时，不生成影子参数候选。
- 影子候选仅用于后续样本外验证。没有明确的验证集改善和风险不恶化证据时，不得标记为可升级，更不得自动修改 `valuation_trend_v1` 正式模型。

以上结果仅供研究筛选，不构成投资建议或自动交易指令。

## 美股严格时点回测

严格时点回测用于验证 `valuation_trend_v1` 在历史样本中的表现，默认只做 8 个分散周次试跑，不接入正式每周筛选任务，也不得自动升级正式模型。

试跑命令：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_us_point_in_time_backtest.ps1 -PilotWeeks 8 -SecUserAgent "Your Name your-email@example.com"
```

只检查执行计划，不联网、不写入文件：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_us_point_in_time_backtest.ps1 -PilotWeeks 8 -SecUserAgent "Your Name your-email@example.com" -DryRun
```


主要输出目录为 `outputs/backtests/us_3y_weekly`，关键文件包括：

```text
outputs/backtests/us_3y_weekly/historical_membership.csv
outputs/backtests/us_3y_weekly/replay_manifest.csv
outputs/backtests/us_3y_weekly/checkpoint.json

outputs/backtests/us_3y_weekly/data_leakage_audit.md
outputs/automation/latest_backtest_summary.md
```

当 `historical_membership.csv` 已经存在，但 `price_history.csv` 或 `benchmark_history.csv` 缺失或为空时，脚本会自动调用 `backtest_price_inputs.py` 准备 5 年股票历史行情和标普 500 基准历史行情；如果覆盖率低于 80%，不会覆盖已有正式输出。
当 `historical_membership.csv` 缺失或为空时，脚本会先调用 `backtest_membership_inputs.py` 基于本地 `us_universe_symbols.csv` 生成带 CIK 的周度成员名单；该名单证据等级为 `secondary`，只用于机械试跑，不得作为模型升级依据。
历史成分证据包可放在 `data/config/us_sp500_membership_evidence.csv`。字段为 `effective_date, added_ticker, removed_ticker, membership_evidence, membership_source_url, notes`。只有官方 S&P Global HTTPS 来源可以保留 `verified`；非官方来源即使写入 `verified` 也会自动降级为 `secondary`。证据包缺失时，脚本仍按当前股票池生成 `secondary` 名单，仅用于机械试跑。
`backtest_report.md` 会汇总本次回放窗口的成员证据覆盖情况，包括 `verified` 比例、`secondary` 数量、`insufficient` 数量、弱证据行数和存在弱证据的回放周数；该信息用于判断历史证据是否足以支持后续模型升级。
`outputs/automation/latest_backtest_summary.md` 是严格时点回测的短摘要入口，记录完成周数、失败周数、成员证据覆盖、弱证据行数、回测报告、泄漏审计、模型对比和日志路径。
当 `data/cache/sec_companyfacts` 缺失或为空时，脚本会调用 `backtest_sec_cache.py` 按成员名单中的 CIK 准备 SEC Company Facts 缓存；如果覆盖率低于 80%，回测会停止，避免用严重缺失的财务事实继续生成结论。
需要先做小样本真实联调时，可临时传入 `-MaxCompanies 3` 等参数限制准备阶段的公司数量；默认值为 0，表示正式运行不限制公司数量。

质量门要求：

- 历史成分证据必须区分 `verified`、`secondary`、`insufficient`，二手或证据不足周次不得作为升级依据。
- 单周回放只能使用 `generated_date` 当日已经可用的成分、SEC 申报和行情数据；`data_leakage_audit.md` 出现严重泄漏时，该周不可用于模型升级判断。
- `backtest_forecasts.csv` 以市场、股票、生成日、模型版本为唯一键；`backtest_evaluations.csv` 还要加检查点周数和评价版本去重。
- `model_comparison.csv` 只比较正式模型和影子参数方案；结论即使出现 `review_candidate`，也只能进入人工复核，不得自动升级正式模型。
