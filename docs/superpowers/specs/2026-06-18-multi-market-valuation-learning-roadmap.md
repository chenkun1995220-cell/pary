# 多市场估值、预测跟踪与自我优化路线图

## 1. 目标与范围

在现有标普 500 低估筛选系统上，按四个阶段扩展：

1. 增加沪深 300、恒生综合大型股指数和中型股指数成分股。
2. 为候选公司输出建议关注价区间、建议买入价区间和 12 个月目标价。
3. 跟踪候选公司实际走势，并与 4、12、26、52 周预测对比。
4. 每周分析模型误差、市场偏差和参数稳定性，在影子模型中寻找可优化逻辑和参数。

系统输出是基于公开数据和模型假设的估值参考，不构成个性化投资建议，也不承诺价格预测实现。

## 2. 总体数据架构

采用“底层分开、标准层统一、评分层部分共享”的分层架构。

### 2.1 市场原始数据层

美股、A股、港股分别设置数据适配器和缓存目录，保留原始代码、原始字段、来源、抓取时间、财报口径和币种。不同市场的接口故障、字段变化和交易规则不会直接污染其他市场。

股票池范围：

- 美股：标普 500 成分股的全部证券行。
- A股：沪深 300 成分股。
- 港股：恒生综合大型股指数和中型股指数成分股，并过滤长期停牌、流动性不足和财务数据严重缺失的公司。

所有成分股名单使用最后一次有效缓存。在线名单必须通过行数、唯一代码、必填字段和来源日期校验后才能替换缓存。

### 2.2 统一标准数据层

三个市场统一使用以下核心字段：

```text
market, ticker, raw_ticker, company_name, industry
currency, accounting_standard, report_date, price_date
market_cap, enterprise_value, net_debt
revenue_ttm, net_income_ttm, operating_cash_flow, capex, free_cash_flow
pe, pb, ev_ebitda, fcf_yield
roe, roic, gross_margin, current_ratio
revenue_cagr_3y, net_income_cagr_3y
raw_source, source_updated_at, data_quality_status
```

金额字段保留原币值和统一比较币值。跨市场汇总时记录汇率日期，不使用未来汇率回填历史数据。

### 2.3 指标和评分层

通用指标计算逻辑共享，估值参数和市场规则分开：

- 通用指标：盈利能力、现金流、成长、资产负债、估值折价和数据质量。
- 美股特有项：SEC 申报完整性、股份类别和美元利率环境。
- A股特有项：ST、停牌、涨跌停、商誉、股权质押和监管风险。
- 港股特有项：成交活跃度、汇率、AH 股溢价、公司行动和长期停牌风险。

行业中位数按“市场 + 标准行业”分别计算。核心评分框架可以共享，但三个市场分别维护参数，不直接共用同一估值中枢。

## 3. 阶段一：A股和港股扩展

### 3.1 数据流程

```text
市场成分股名单
  -> 市场专用公司、行情和财报适配器
  -> 原始数据缓存
  -> 统一字段映射与单位标准化
  -> 市场数据质量检查
  -> 市场行业中位数
  -> 候选评分与统一报告
```

每个市场可以独立失败和重跑。某一市场数据源失败时，其他市场仍可产出报告；失败市场明确标记为缓存数据或本期不可用。

### 3.2 验收标准

- 三个股票池均可自动刷新并缓存回退。
- 股票代码、币种、报告期和单位可以稳定标准化。
- 各市场成分股匹配率不低于 98%。
- 严重数据缺失公司不能进入候选池。
- 周报能分别展示三个市场结果，并提供统一候选汇总。

## 4. 阶段二：建议买入价和目标价

### 4.1 估值模型

每家公司同时计算三类估值：

1. 现金流或盈利折现模型。
2. 公司自身历史估值区间。
3. 同市场、同行业相对估值。

模型根据行业和数据完整性分配权重。金融、地产等不适合通用企业价值模型的行业使用专用估值模板。

### 4.2 价格输出

- 建议关注价：接近合理价值且具备初步安全边际的价格区间。
- 建议买入价：满足更严格安全边际的价格区间，不输出单一点位伪精确值。
- 12 个月目标价：悲观、基准、乐观三种情景。
- 预期空间：相对当前价格的上涨和下跌空间。
- 置信度：由数据质量、模型一致性、盈利稳定性和估值敏感度共同决定。

报告必须列出关键假设，包括收入增长、利润率、资本成本、终值、行业估值中枢和汇率，并展示主要参数变化对目标价的影响。

### 4.3 走势判断

走势判断不是单纯价格拟合，而是估值、盈利修正、动量、波动率、成交活跃度和市场环境的组合。分别输出未来 4、12、26、52 周的方向、预期区间、失效条件和主要催化因素。

## 5. 阶段三：预测跟踪与误差对比

每周生成不可修改的预测快照，保存当时可获得的数据、模型版本、参数、价格区间、目标价、方向、置信度和风险提示。后续不得用新财报或新行情改写历史预测。

到达 4、12、26、52 周评价点时，计算：

- 方向命中率。
- 目标价和区间误差。
- 最大有利和最大不利波动。
- 相对基准指数的超额收益。
- 按市场、行业、评分等级和置信度分组的表现。

公司拆股、分红、配股、合并和停牌等事件必须进行复权或单独标记，避免把公司行动误判为预测错误。

## 6. 阶段四：每周自我分析与影子模型

每周任务在正式报告完成后执行模型审计：

1. 检查评分因子与后续收益的关系是否减弱。
2. 检查市场、行业、规模和风格偏差。
3. 分析目标价误差来源和置信度校准情况。
4. 搜索候选参数组合和可解释的新逻辑。
5. 在影子模型中进行滚动回测和样本外比较。

正式模型不得自动修改。影子模型只有同时满足以下条件时才能生成升级建议：

- 样本外表现优于正式模型。
- 改进不依赖单一市场、行业或少数公司。
- 最大回撤和尾部风险没有明显恶化。
- 参数在多个滚动窗口中保持稳定。
- 没有使用预测时点之后的数据。

升级建议报告必须展示旧模型、新模型、变化原因、训练区间、验证区间、收益和风险差异。只有人工确认后才能提升为正式模型，并保留回滚版本。

## 7. 每周任务顺序

```text
刷新三市场成分股和数据
  -> 数据质量检查
  -> 正式模型筛选和估值报告
  -> 保存本周预测快照
  -> 更新历史预测实际表现
  -> 正式模型与影子模型对比
  -> 生成模型审计和优化建议
```

任何抓取失败、缓存回退、模型版本变化和历史数据修订都需要进入当周运行摘要。

## 8. 实施顺序

后续实施按四个独立项目推进，每个项目单独设计、测试和验收：

1. 沪深 300 与港股大中盘数据接入。
2. 多模型估值、买入区间和目标价。
3. 预测快照、实际走势跟踪和误差分析。
4. 影子模型、滚动回测和每周优化审计。

在前三阶段没有形成足够历史预测样本之前，第四阶段只做数据质量和逻辑审计，不根据少量样本调整正式参数。
