# 预测跟踪与模型审计设计

## 目标

在三市场每周筛选和12个月估值之后，持续跟踪每条历史预测的实际表现，在4、12、26、52周评价点生成不可修改的正式成绩，并每周输出模型偏差分析与影子模型建议。

正式模型参数不得自动改变。所有优化结论必须来自当时可获得的数据、时间顺序明确的样本外验证和人工确认。

## 范围

本阶段包含：

- 历史预测的每周实际表现更新。
- 股票绝对收益和相对市场基准的超额收益。
- 4、12、26、52周正式评价。
- 方向命中、目标价误差、最大有利和最大不利波动。
- 按市场、行业、评分和置信度分组的模型审计。
- 样本满足门槛后的影子参数建议。
- 接入美股、A股和港股三条周任务。

本阶段不包含：

- 自动修改正式评分或估值参数。
- 自动下单、仓位优化和组合交易成本。
- 使用未来数据回填或改写历史预测。
- 在样本不足时生成参数优劣结论。

## 模块结构

### 预测跟踪器

预测跟踪器读取不可修改的 `forecast_history.csv`，合并股票复权行情和所属市场基准行情，输出每周跟踪状态与到期评价。

职责包括：

- 为每条预测找到生成日当天或之前最近交易日的起始复权价。
- 为本周跟踪日和正式评价日找到当天或之前最近交易日价格。
- 计算绝对收益、基准收益、超额收益、目标价误差、最大有利波动和最大不利波动。
- 在4、12、26、52周评价点写入唯一、不可覆盖的评价记录。
- 标记公司行动、停牌、行情缺失和基准缺失。

### 模型审计器

模型审计器只读取预测、跟踪和正式评价结果，不直接调用或修改正式评分模型。

职责包括：

- 汇总方向命中率、收益、超额收益、误差和尾部风险。
- 按市场、行业、评分等级、置信度和模型版本分组。
- 检查置信度是否与实际命中率匹配。
- 在满足最小样本门槛时生成影子参数方案与样本外比较。
- 输出升级建议或“样本积累中”，不修改正式配置。

## 行情与市场基准

跟踪计算使用复权收盘价。行情适配层需要保存原始收盘价、复权收盘价、分红和拆股事件；无法确认复权或出现异常跳变时，相关评价标记为 `corporate_action_review`，不进入模型成绩。

默认市场基准：

- 美股：标普500指数。
- A股：沪深300指数。
- 港股：恒生综合指数；若数据源不可用，则本期港股超额收益标记为基准缺失，不得静默替换为其他指数。

基准代码保存在独立配置中，数据源代码在实施时需要真实联网验证。基准历史与股票历史使用相同日期和复权规则。

## 评价周期

每周更新所有仍在跟踪中的预测。正式评价点为预测生成日后的：

- 4周：28个自然日。
- 12周：84个自然日。
- 26周：182个自然日。
- 52周：364个自然日。

正式评价取评价日当天或之前最近交易日。若最近有效交易日与评价日相差超过7个自然日，该评价标记为 `insufficient_data`，等待后续数据修复，不判定成功或失败。

同一预测在同一评价点只允许一条正式记录。后续重跑不得覆盖原评价；数据修订必须产生带修订原因和新版本号的独立记录。

## 评价公式

### 收益与超额收益

- 股票实际收益 = 评价复权价 / 起始复权价 - 1。
- 基准收益 = 基准评价复权价 / 基准起始复权价 - 1。
- 超额收益 = 股票实际收益 - 基准收益。

### 方向

根据预测时的 `expected_return` 固定方向：

- 大于 `+5%`：看涨。
- 小于 `-5%`：看跌。
- `-5%` 至 `+5%`：中性。

实际方向使用同样的5%阈值。预测方向和实际方向相同时记为命中。阈值属于评价口径版本的一部分，历史记录不得随未来参数调整而重算。

### 目标价误差

- 目标价绝对误差 = 实际评价价 - 预测目标价。
- 目标价百分比误差 = `abs(实际评价价 - 预测目标价) / 预测目标价`。
- 52周评价是12个月目标价的最终误差；4、12、26周只作为路径观察，不提前判定12个月目标价最终成功或失败。

### 路径风险

- 最大有利波动 = 跟踪区间内最高复权价 / 起始复权价 - 1。
- 最大不利波动 = 跟踪区间内最低复权价 / 起始复权价 - 1。

每周跟踪记录显示当前状态，但未到评价点时统一标记为 `tracking`。

## 输出

每个市场输出目录新增：

- `tracking_snapshot.csv`：所有历史预测截至本周的实际表现。
- `forecast_evaluations.csv`：4、12、26、52周正式评价历史。
- `performance_report.md`：中文表现报告。
- `model_audit.md`：中文模型偏差、稳定性和数据质量审计。
- `shadow_model_proposals.csv`：影子参数方案、训练区间、验证区间和比较结果。

正式评价唯一键为“市场、股票代码、预测生成日期、预测模型版本、评价周数、评价口径版本”。每周跟踪快照可以原子替换，预测历史和正式评价只允许追加与去重。

## 模型审计与影子模型

每周模型审计必须先报告：

- 成熟评价样本数和仍在跟踪的样本数。
- 行情、基准和公司行动异常数。
- 按市场、行业、评分等级和置信度的样本分布。
- 方向命中率、平均收益、平均超额收益、目标价误差和最大不利波动。

影子模型规则：

- 成熟评价样本少于30条时，只输出数据质量和分布，不提出参数升级。
- 使用按预测时间排序的训练集和验证集，不得随机打乱时间。
- 样本外验证记录少于15条时，不形成升级建议。
- 改进不得只来自单一市场、单一行业或少数公司。
- 最大不利波动和尾部误差不得明显恶化。
- 参数需要在多个滚动窗口中方向一致。
- 所有候选参数、训练区间、验证区间和指标必须写入影子结果。

正式模型升级只能由人工确认。系统不得写回正式参数文件、模型版本或历史预测。

## 失败处理

- 股票或基准行情覆盖率低于80%时，本期跟踪步骤失败，不覆盖旧正式报告。
- 基准缺失时保留股票绝对收益，但超额收益为空并记录警告。
- 复权异常、拆股、分红、合并、配股或停牌无法确认时，不计入模型成绩。
- 预测起始价缺失时整条预测保持 `insufficient_data`。
- 尚未到正式评价点时不得生成到期评价。
- 抓取失败、缓存回退、模型版本变化和数据修订必须进入运行摘要。

## 周任务顺序

每个市场的周任务顺序扩展为：

1. 刷新成分股、财务和行情。
2. 运行正式筛选与12个月估值。
3. 追加本周预测快照。
4. 更新历史预测实际表现。
5. 写入到期的4、12、26、52周评价。
6. 运行正式模型审计和影子模型比较。
7. 生成中文报告与运行摘要。

任何评价或审计失败不得把旧报告标记为本周新结果。

## 测试与验收

自动化测试覆盖：

- 复权行情解析、拆股分红事件和异常标记。
- 起始日、评价日和最近交易日选择。
- 4、12、26、52周到期判断和7天容差。
- 绝对收益、基准收益、超额收益和目标价误差。
- 最大有利、最大不利波动和方向命中。
- 未到期预测只标记 `tracking`。
- 正式评价唯一键去重和不可覆盖。
- 基准缺失、行情覆盖不足和公司行动异常。
- 分组审计与置信度校准。
- 30条成熟样本、15条样本外记录和时间顺序切分门槛。
- 正式模型文件永不被审计器修改。
- 三市场周脚本的调用顺序与失败传播。

真实验收需要在当前三市场预测历史上运行。由于首批预测刚生成，首次报告应明确显示“样本积累中”，不得生成参数升级建议；同时验证三条自动任务仍为启用状态并读取新跟踪产物。
