# 每周自我分析摘要设计

## 背景

项目已经拆分出美股、A股、港股三个每周筛选任务，并新增了美股严格时点回测摘要。当前的问题是：这些结果分散在不同目录，自动任务完成后缺少一个统一入口来判断“本周是否跑完、哪里有缺口、下一步优先优化什么”。

## 目标

新增统一自我分析摘要，输出到：

`outputs/automation/latest_self_analysis.md`

该文件汇总：

- 美股、A股、港股周筛摘要是否存在。
- 三市场候选公司数量和候选代码。
- 三市场模型审计文件路径和审计状态。
- 美股严格时点回测完成周数、失败周数、成员证据覆盖和弱证据行数。
- 缺失摘要、失败周、弱证据和样本积累不足等风险。
- 下周优先优化建议。

## 非目标

- 不自动修改 `valuation_trend_v1` 正式模型。
- 不重新运行三市场周筛或严格时点回测。
- 不读取或改写本地行情样本文件 `data/samples/us_universe_quotes.csv`。
- 不替代各市场原始周报、估值报告或回测报告。

## 组件

### Python 汇总器

新增 `automation_self_analysis.py`：

- 读取固定路径下的摘要和审计文件。
- 解析 Markdown 中的 `- Key: Value` 或 `- Key：Value` 行。
- 生成中文 Markdown 自我分析摘要。
- 暴露 CLI 参数 `--project-root`、`--output`、`--as-of-date`。

### PowerShell 入口

新增 `scripts/run_self_analysis.ps1`：

- 默认使用当前项目根目录。
- 支持 `-Output` 和 `-DryRun`。
- 非 dry-run 时调用 `automation_self_analysis.py`。

## 输出格式

摘要包含四段：

1. `运行覆盖`：三市场周筛状态、候选数量、审计状态。
2. `严格时点回测`：回测摘要状态、完成周数、失败周数、证据覆盖。
3. `风险与缺口`：缺失摘要、失败周、弱证据、审计样本积累等问题。
4. `下周优化建议`：按风险生成的保守建议。

## 错误处理

缺失输入文件不视为程序失败，而是在自我分析摘要中标记为 `missing`。只有输出路径无法写入或输入路径参数非法时，脚本失败。

## 测试

- 单元测试构造临时项目目录，写入三市场摘要、模型审计和回测摘要，确认输出包含候选数量、审计状态、证据覆盖和优化建议。
- 单元测试缺失摘要场景，确认输出标记缺失并给出补齐建议。
- 脚本契约测试确认 `run_self_analysis.ps1` 存在 dry-run、目标输出和 Python 汇总器调用。
