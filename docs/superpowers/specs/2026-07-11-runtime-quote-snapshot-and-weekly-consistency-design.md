# 行情快照与周交付一致性设计

## 目标

完成中期目标 P5：让每周行情刷新不再长期污染 Git 工作区，同时让提交前复核能够证明本周三市场日期、候选数量和统一交付结论彼此一致。

## 现状与问题

`data/samples/us_universe_quotes.csv` 当前由 Git 跟踪，却又被美股周任务原地覆盖。刷新器会重写全部 503 行，因此即使大部分价格仍复用旧值，工作区也会产生整表差异。该文件属于可重建的运行快照，不是人工维护的配置或稳定测试夹具，不适合按周进入版本历史。

现有周收口会重跑提交前复核，但提交前复核主要检查各自动化 JSON 的结构与新鲜度，尚未直接证明三市场运行摘要、候选池、统一结论和交付验收的候选数一致。

## 采用方案

### 1. 行情快照改为运行产物

- 美股周任务将行情快照写入 `outputs/us_universe/market_quotes.csv`。
- `outputs/` 已由 `.gitignore` 忽略，因此每周刷新不会形成 Git 差异。
- 删除 Git 跟踪的 `data/samples/us_universe_quotes.csv`；不使用 `skip-worktree` 等只在单机生效且会隐藏真实变化的机制。
- 手工命令仍可显式传入其他 `-Output` 或 `-Quotes` 路径，保持调试与夹具测试能力。
- 美股运行摘要记录快照相对路径、行数、最早/最新行情日期和 SHA-256，形成可审计的当次输入证据。

### 2. 独立的一致性复核产物

新增周产物一致性复核，输出：

- `outputs/automation/latest_weekly_artifact_consistency.json`
- `outputs/automation/latest_weekly_artifact_consistency.md`

复核读取三市场 `latest_run_summary.md`、`candidate_pool.csv`、统一周结论和交付验收，不联网、不重新评分。它检查：

- 三市场运行摘要日期均在允许的新鲜度窗口内、不得位于未来，并且必须属于同一自然日；14:05、14:10、14:15 的具体运行时刻可以不同。
- 每个市场摘要中的候选数等于对应 `candidate_pool.csv` 行数。
- 统一周结论中的分市场候选数等于对应候选池行数，总数等于三市场之和。
- 交付验收候选总数等于统一周结论总数。
- 统一周结论和交付验收的 `as_of_date` 相同。
- 美股运行摘要声明 `runtime_output_only` 快照策略，快照路径位于 `outputs/us_universe`，且摘要中的行数、日期范围和 SHA-256 与实际文件一致。
- 旧路径 `data/samples/us_universe_quotes.csv` 不存在于当前工作树。

任一检查失败时，状态为 `needs_attention`，并输出明确问题码和对应路径；不存在“只提示但仍标记 ready”的软通过。

### 3. 周任务和提交前复核接入

- `run_weekly_reporting_bundle.ps1` 在统一周结论和交付验收完成后运行一致性复核，再运行提交前复核。
- `pre_submit_review.py` 把一致性复核作为必需且需要新鲜的输入。
- 一致性复核不是行情抓取步骤，不会改变候选结果；它只证明刚生成的产物属于同一个有效周收口。
- 美股、A股、港股使用 `-RunPostChecks` 时继续自动执行最终提交前复核。

## Git 与迁移边界

- 仓库保留行情生成代码、人工股本覆盖、公司名单、测试夹具和复核规则。
- 仓库不保留每周可重建的全量美股行情快照。
- 主工作区当前未提交的 `data/samples/us_universe_quotes.csv` 只作为迁移前运行数据；迁移后由新的运行快照取代，不将该整表刷新提交到 Git。
- 不触碰与本任务无关的本地文档改动。

## 测试与验收

- 单元测试覆盖快照路径、行数、日期、哈希、候选数和日期不一致的失败场景。
- 脚本测试确认一致性复核位于交付验收之后、提交前复核之前。
- 提交前复核测试确认一致性产物缺失、过期或 `needs_attention` 时不得 `ready`。
- 在隔离工作区运行完整测试。
- 合并后使用当前三市场产物生成一次一致性复核和提交前复核，要求两者均为 `ready`。
- 最终 `git status` 不再因每周行情整表刷新而出现 `data/samples/us_universe_quotes.csv`。

## 安全边界

- 不自动提交或推送运行数据。
- 不修改正式估值、筛选或预测模型参数。
- 不把缺失或过期快照伪装为可用。
- 不要求三个独立市场任务具有完全相同的运行时刻，但要求运行日期属于同一自然日、各自新鲜且候选与统一交付一致。
