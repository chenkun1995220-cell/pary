# 关键周度 JSON 原子写入设计

## 背景

关键周度 JSON 由多个独立脚本直接覆盖写入。进程中断时可能留下半截文件；
同时，无 BOM 的 UTF-8 JSON 在 Windows PowerShell 5 默认读取路径下可能被错误
解码，造成“文件损坏”的误报。

## 设计

- 新增共享函数 `write_json_atomic(path, payload)`。
- 先在目标目录写入同盘临时文件，再用原子替换发布。
- 临时文件使用 `utf-8-sig`，确保 Python 与 Windows PowerShell 5 均可读取。
- 替换失败时保留原文件并清理临时文件。
- 接入三市场完成屏障、一致性检查、交付检查、提交前复核和人工处理清单。
- 历史 JSONL 保持追加写入，不纳入本次修改。

## 安全边界

本改动只改变产物发布方式和文本编码，不改变数据、筛选、估值、预测或质量
判断逻辑。
