# 2026-07-19 首批一个月预测成熟提示设计

## 目标

让每周人工处理清单同时显示两个不同的一个月预测评价节点，避免只看到下一批
全市场评价日期而错过更早成熟的首批固定队列：

- 首批固定队列：2026-08-03，37 条港股样本。
- 下一批全市场队列：2026-08-15，62 条三市场样本。

## 推荐方案

沿用现有数据流，不让待办生成器直接读取额外产物：

1. `automation_self_analysis.py` 从
   `latest_first_one_month_forecast_evaluation_review.json` 的固定队列和一个月评价
   字段中提取成熟日期与预期样本数。
2. 将上述字段写入 `latest_self_analysis_manifest.json` 的
   `first_one_month_forecast_evaluation` 摘要。
3. `weekly_action_items.py` 在 `continue_sample_accumulation` 的来源和建议说明中
   同时展示首批固定队列与下一批全市场队列。

这种方式保持自我分析 manifest 作为待办生成器的单一输入契约，避免引入新的文件耦合。

## 显示与安全边界

- 只有首批复核状态为 `awaiting_maturity` 且成熟日期存在时才显示首批检查点。
- 首批与下一批日期分别标注，不合并样本数，也不把重复运行计为新批次。
- 缺失首批成熟日期时继续使用现有下一批提示，不制造日期。
- 提示仅用于安排评价时间，不改变预测、选股、估值或正式模型参数。
- 两批评价均保留 `formal_model_change_allowed=false` 边界。

## 验证

- 先增加自我分析摘要测试，确认成熟日期和固定队列样本数能够进入 manifest。
- 增加待办生成测试，确认建议同时包含两个日期和两组样本数。
- 先确认新增测试在现有实现上失败，再实施最小修复。
- 运行相关测试、全量测试和一次只读待办报告生成。
