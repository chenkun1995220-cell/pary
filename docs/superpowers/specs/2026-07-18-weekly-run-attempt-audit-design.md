# 周度运行尝试审计设计

## 目标

区分自动化计划窗口内的首次运行与失败后的修复运行，避免最终摘要覆盖首次失败
证据。

## 设计

- 每次市场脚本写入 `running`、`ready` 或 `failed` 状态时，同时追加状态历史。
- 同一市场和 `run_started_at` 组成稳定的 `attempt_id`。
- 一致性产物按日期归并尝试，输出首次、最后一次、尝试数和每次最终状态。
- 港股额外输出是否存在 14:30–14:45 窗口内尝试，以及是否检测到修复运行。
- 连续周六验收继续按最终成功摘要的启动时间执行原有门槛，不因存在早期失败
  尝试而放宽。
- 新增独立的“计划窗口尝试已观察”状态，供诊断使用。

## 边界

不重新运行抓取，不修改正式模型，不回填或猜测部署前的历史尝试。
