# 周度子进程瞬时失败重试设计

## 背景

2026-07-18 的实际运行中，A 股行情快照和港股行情缺口诊断曾以退出码
`120` 失败，完整重跑后成功。现有 Python 抓取器已包含请求级重试，但周度
PowerShell 编排层没有处理这种子进程级瞬时失败。

## 设计

- 新增共享 PowerShell 函数 `Invoke-WeeklyTransientStep`。
- 默认只把退出码 `120` 视为可重试错误。
- 最多执行两次，第二次仍失败时保留步骤名称和最终退出码并终止链路。
- 退出码不是 `120` 时立即终止，不做重试。
- 首批只接入本周有直接失败证据的行情快照和两次行情缺口诊断。
- 不接入筛选、估值、预测和报告等本地计算步骤。
- 不改变 Python 抓取器的覆盖率、数据新鲜度和禁止缓存回退门槛。

## 验收

- 首次返回 `120`、第二次成功时，步骤整体成功且只执行两次。
- 非 `120` 错误只执行一次并原样失败。
- 连续两次返回 `120` 时，步骤失败并报告第二次退出码。
- A 股和港股 DryRun 仍不产生文件或网络请求。
