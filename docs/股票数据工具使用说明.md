# 股票低估筛选数据工具使用说明

> 版本：2026-06-16  
> 位置：`F:\chatgptssd\project2`

## 1. 工具目标

这个工具是第 3 步的第一版本地骨架：把美股、港股、A股的标准化 CSV 数据放入同一个目录后，自动合并、计算估值指标、按前一版筛选标准打分，并输出完整结果和候选池。

第一版重点不是联网抓取全市场数据，而是先把“数据整合 -> 标准化 -> 打分 -> 导出候选池”的主流程跑通。后续可以逐步接入 SEC EDGAR、HKEXnews、巨潮资讯、上交所、深交所等数据源。

## 2. 文件结构

| 路径 | 用途 |
|---|---|
| `stock_screener.py` | 主程序，负责读取 CSV、评分、导出 |
| `data/raw/sample_stocks.csv` | 示例输入数据 |
| `outputs/screening_results.csv` | 完整打分结果 |
| `outputs/candidate_pool.csv` | 候选池结果，默认总分不低于 80 且无重大风险 |
| `tests/test_stock_screener.py` | 自动化测试 |

## 3. 输入 CSV 字段

必填或建议字段如下。字段名必须使用英文表头，便于后续自动化处理。

| 字段 | 含义 | 备注 |
|---|---|---|
| `market` | 市场 | 美股、港股、A股 |
| `ticker` | 股票代码 | 如 `AAPL`、`0700.HK`、`600519` |
| `company_name` | 公司名称 | 中文或英文均可 |
| `industry` | 行业 | 建议使用统一行业分类 |
| `market_cap` | 总市值 | 与其他金额字段保持同一货币 |
| `enterprise_value` | 企业价值 EV | 市值 + 有息债务 - 现金等 |
| `net_assets` | 净资产 | 用于计算 PB |
| `revenue_ttm` | 最近 12 个月收入 | TTM 优先 |
| `net_income_ttm` | 最近 12 个月净利润 | 用于 PE、净利率 |
| `ebitda` | EBITDA | 用于 EV/EBITDA |
| `operating_cash_flow` | 经营现金流 | 用于现金流质量 |
| `capex` | 资本开支 | 建议用负数录入 |
| `dividend_yield` | 股息率 | 小数形式，如 5% 录入 `0.05` |
| `industry_pe_median` | 行业 PE 中位数 | 同市场、同行业、同商业模式 |
| `industry_pb_median` | 行业 PB 中位数 | 同上 |
| `industry_ev_ebitda_median` | 行业 EV/EBITDA 中位数 | 金融行业可留空 |
| `roe` | ROE | 小数形式 |
| `roic` | ROIC | 小数形式 |
| `gross_margin` | 毛利率 | 小数形式 |
| `debt_to_assets` | 资产负债率 | 小数形式 |
| `net_debt_to_ebitda` | 净负债/EBITDA | 净现金公司可录入 0 或负数 |
| `current_ratio` | 流动比率 | 数值 |
| `revenue_cagr_3y` | 3 年收入 CAGR | 小数形式 |
| `net_income_cagr_3y` | 3 年净利 CAGR | 小数形式 |
| `audit_opinion` | 审计意见 | 建议使用 `标准无保留` 等固定值 |
| `risk_flag` | 风险标记 | `无`、`轻微`、`重大` |

## 4. 运行方式

在项目目录运行：

```powershell
& 'C:\Users\pechen\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe' stock_screener.py --raw-dir data/raw --output-dir outputs
```

默认规则：

1. 读取 `data/raw` 目录下所有 `.csv` 文件；
2. 输出完整结果到 `outputs/screening_results.csv`；
3. 输出候选池到 `outputs/candidate_pool.csv`；
4. 候选池默认要求总分不低于 80，且 `risk_flag` 不能是 `重大`。

生成每周报告：

```powershell
.\scripts\run_weekly_screening.ps1
```

运行后会额外生成：

| 输出 | 说明 |
|---|---|
| `outputs/reports/YYYY-MM-DD_低估公司每周筛选报告.md` | 当周低估候选报告，包含候选公司、评分和理由 |
| `outputs/screening_results.csv` | 全量打分结果 |
| `outputs/candidate_pool.csv` | 候选池 |

如果想调低候选池门槛，例如 70 分：

```powershell
& 'C:\Users\pechen\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe' stock_screener.py --raw-dir data/raw --output-dir outputs --candidate-min-score 70
```

## 9. SEC 美股数据接入 V1

当前已经接入第一版 SEC Company Facts 适配器：

| 文件 | 用途 |
|---|---|
| `sec_edgar_adapter.py` | 读取 SEC Company Facts JSON，并转换为标准股票筛选 CSV |
| `data/config/sec_us_companies.csv` | 美股公司配置表，填写 ticker、CIK、行业、市值、行业中位数等 |
| `scripts/import_sec_us_stocks.ps1` | 单独刷新 SEC 美股数据 |
| `scripts/run_weekly_screening.ps1` | 每周运行脚本；如果检测到 `SEC_USER_AGENT`，会先刷新 SEC 数据 |

SEC 官方接口：

```text
https://data.sec.gov/api/xbrl/companyfacts/CIK##########.json
```

其中 `##########` 是补足 10 位的 CIK。SEC 官方说明显示，该接口返回单家公司所有 Company Facts，一次请求即可拿到多项 XBRL 财务概念；接口不需要 API key，但自动访问需要遵守 SEC 访问政策，并提供合适的 User-Agent。

### 9.1 配置美股公司

编辑：

```text
data/config/sec_us_companies.csv
```

建议至少填写：

| 字段 | 说明 |
|---|---|
| `ticker` | 股票代码 |
| `cik` | SEC CIK，可不带前导 0 |
| `company_name` | 公司名称 |
| `industry` | 行业 |
| `market_cap` | 总市值，SEC 财报不提供，需要外部行情数据补充 |
| `enterprise_value` | 企业价值 EV，需要外部行情和债务现金数据补充 |
| `industry_pe_median` | 行业 PE 中位数 |
| `industry_pb_median` | 行业 PB 中位数 |
| `industry_ev_ebitda_median` | 行业 EV/EBITDA 中位数 |

SEC V1 自动提取：

| 标准字段 | SEC XBRL 概念候选 |
|---|---|
| `revenue_ttm` | `Revenues`、`RevenueFromContractWithCustomerExcludingAssessedTax`、`SalesRevenueNet` |
| `net_income_ttm` | `NetIncomeLoss`、`ProfitLoss` |
| `net_assets` | `StockholdersEquity` 等 |
| `operating_cash_flow` | `NetCashProvidedByUsedInOperatingActivities` |
| `capex` | `PaymentsToAcquirePropertyPlantAndEquipment`，会转为负数 |
| `debt_to_assets` | `Liabilities / Assets` |

### 9.2 手动刷新 SEC 数据


然后运行：

```powershell

```

输出文件：


这个文件会被现有筛选器自动读取。

### 9.3 每周自动刷新

如果 Windows 任务计划程序运行环境里设置了 `SEC_USER_AGENT`，每周脚本会自动执行：

1. 从 SEC 刷新 `data/raw/sec_us_stocks.csv`；
2. 合并 `data/raw` 下所有 CSV；
3. 输出 `screening_results.csv`、`candidate_pool.csv`；
4. 生成带日期的每周 Markdown 报告。

### 9.4 当前限制

1. SEC 财报数据本身不提供实时股价、市值、行业中位数，这些仍需配置或后续接行情数据源；
2. 第一版优先选择最新年度 `10-K`、`20-F`、`40-F` 数据，还不是完整 TTM 滚动口径；
3. 不同公司 XBRL 标签可能有差异，已覆盖常见概念，但后续需要根据真实样本扩展；
4. 港股和 A股尚未接入官方抓取模块。

| 文件 | 用途 |

### 10.1 补充字段

编辑：

```text
data/config/us_market_quotes.csv
```

字段说明：

| 字段 | 说明 |
|---|---|
| `ticker` | 股票代码，需与 SEC 配置表一致 |
| `price` | 最新或指定日期收盘价 |
| `shares_outstanding` | 稀释或普通股股本，单位需和价格匹配 |
| `net_debt` | 净债务，等于有息债务减现金；净现金可填负数 |
| `currency` | 货币，通常为 USD |
| `quote_date` | 行情日期 |

计算规则：

```text
market_cap = price * shares_outstanding
enterprise_value = market_cap + net_debt
```

注意单位必须一致。例如价格是美元、股本是百万股，则市值会是百万美元；其他财务字段也应保持同一单位。

### 10.3 每周脚本中的处理方式

每周脚本现在按以下顺序运行：

1. 如果存在 `SEC_USER_AGENT`，先刷新 `data/raw/sec_us_stocks.csv`；
2. 如果存在 `sec_us_stocks.csv` 和 `us_market_quotes.csv`，生成 `data/raw/us_stocks_enriched.csv`；

### 10.4 Stooq 延迟行情入口

代码中已经预留 `fetch_stooq_quote()`，可解析 Stooq 延迟行情 CSV，例如收盘价和日期。当前没有直接把它接入每周脚本，原因是：

1. Stooq 不是官方交易所源；
2. 自动化大量访问需要额外节流和错误处理；
3. 即使能拿到价格，也仍然需要股本和净债务数据才能计算市值和 EV。

因此 V1 先采用 `us_market_quotes.csv` 作为稳定补充层。后续可以增加“自动拉价格 + 本地股本/净债务配置”的混合模式。

## 11. 行业中位数自动维护层 V1

当前已经增加行业中位数自动维护层，用于减少手工填写 `industry_pe_median`、`industry_pb_median`、`industry_ev_ebitda_median`。

| 文件 | 用途 |
|---|---|
| `industry_medians.py` | 按市场和行业计算估值中位数并回填 |
| `scripts/update_industry_medians.ps1` | 单独执行行业中位数更新 |
| `data/derived/industry_medians.csv` | 自动生成的行业中位数表 |
| `data/raw/us_stocks_with_industry_medians.csv` | 回填行业阈值后的美股数据 |

### 11.1 计算口径

行业中位数按 **市场 + 行业** 分组计算，不跨市场混用。例如：

```text
美股 + 科技软件
A股 + 科技软件
港股 + 科技软件
```

会被视为三个不同分组。

计算指标：

```text
PE = market_cap / net_income_ttm
PB = market_cap / net_assets
EV/EBITDA = enterprise_value / ebitda
```

不可比样本会被剔除：

1. 净利润小于等于 0 的样本不参与 PE 中位数；
2. 净资产小于等于 0 的样本不参与 PB 中位数；
3. EBITDA 小于等于 0 的样本不参与 EV/EBITDA 中位数；
4. 缺少市场或行业的样本不参与分组。

### 11.2 回填规则

默认只回填空白字段，不覆盖手工填写的行业阈值。

例如某只股票已有：

```text
industry_pb_median = 99
```

即使自动计算出来的 PB 中位数是 2.5，也不会覆盖 `99`。如需强制覆盖，可以直接运行 Python 命令并加 `--overwrite`。

### 11.3 手动执行

先确保存在：

```text
data/raw/us_stocks_enriched.csv
```

然后运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\update_industry_medians.ps1
```

输出：

```text
data/raw/us_stocks_with_industry_medians.csv
data/derived/industry_medians.csv
```

### 11.4 每周脚本中的处理方式

每周脚本会自动执行行业中位数回填，并在最终筛选输入目录 `data/weekly_raw` 中使用回填后的数据。这样评分时的“相对行业折价”不再完全依赖手工维护。

当前限制：

1. 样本数较少时，中位数代表性不足；输出表里会记录 `sample_count`；
2. 行业分类质量决定中位数质量，后续需要统一行业分类表；
3. 目前只对已有 CSV 数据计算，不会自动下载完整行业成分股。

## 12. 统一行业分类/别名映射层 V1

当前已经增加统一行业分类层，用于解决不同数据源行业名称不一致的问题。

| 文件 | 用途 |
|---|---|
| `industry_mapper.py` | 按别名表统一行业分类 |
| `data/config/industry_aliases.csv` | 行业别名到标准行业的映射表 |
| `scripts/map_industries.ps1` | 单独执行行业映射 |
| `data/raw/us_stocks_industry_mapped.csv` | 行业映射后的美股数据 |

### 12.1 为什么需要这一层

行业中位数依赖 `market + industry` 分组。如果行业字段不统一，例如：

```text
Software
software
SaaS
软件
科技软件
```

它们会被拆成多个分组，导致样本数不足、行业中位数失真。行业映射层会先把这些别名统一到 `科技软件`。

### 12.2 别名表格式

编辑：

```text
data/config/industry_aliases.csv
```

格式：

```text
alias,standard_industry
Software,科技软件
SaaS,科技软件
软件,科技软件
```

匹配规则：

1. 大小写不敏感；
2. 自动去除首尾空格；
3. 多个连续空格会被压缩；
4. 未命中的行业会保留原值，并标记为 `unmapped`。

### 12.3 输出字段

行业映射后会新增：

| 字段 | 说明 |
|---|---|
| `raw_industry` | 原始行业名 |
| `industry` | 标准化后的行业名 |
| `industry_mapping_status` | `mapped` 或 `unmapped` |

这样既能统一分组，也能回查原始来源。

### 12.4 手动执行

先确保存在：

```text
data/raw/us_stocks_enriched.csv
```

然后运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\map_industries.ps1
```

输出：

```text
data/raw/us_stocks_industry_mapped.csv
```

### 12.5 每周脚本中的处理方式

每周脚本现在顺序为：

```text
SEC 财报导入
-> 美股行情/市值补充
-> 行业别名映射
-> 行业中位数计算与回填
-> 低估评分
-> 每周报告
```

当前限制：

1. 行业别名表需要持续维护；
2. 没有自动识别未知行业的语义相似度；
3. 不同行业体系之间的层级关系尚未建模，例如 GICS 一级/二级/三级行业。

| `scripts/map_external_fields.ps1` | 手动转换外部 CSV |
| `data/imports/` | 外部 CSV 放置目录 |

### 13.1 为什么需要字段映射

不同来源的同一字段可能叫法不同：

```text
股票代码 / 证券代码 / 代码 / Ticker
总市值 / 市值 / market_cap
归母净利润TTM / 净利润TTM / 净利润
经营现金流净额 / 经营现金流
```

评分器只识别标准字段，例如：

```text
ticker
market_cap
net_income_ttm
operating_cash_flow
```

字段映射层负责把外部表头转换为这些标准字段。

### 13.2 字段别名表

编辑：

```text
data/config/field_aliases.csv
```

格式：

```text
alias,standard_field
股票代码,ticker
总市值,market_cap
归母净利润TTM,net_income_ttm
经营现金流净额,operating_cash_flow
```

匹配规则：

1. 大小写不敏感；
2. 自动去除首尾空格；
3. 多个连续空格会被压缩；
4. 未识别字段不会丢弃，会加上 `source_` 前缀保留。

### 13.3 手动转换外部 CSV

把外部 CSV 放入：

```text
data/imports/
```

然后运行：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\map_external_fields.ps1 -InputPath F:\chatgptssd\project2\data\imports\your_export.csv -Output data\raw\imported_mapped.csv
```

输出：

```text
data/raw/imported_mapped.csv
```

转换后建议先打开检查：

1. `ticker`、`company_name`、`industry` 是否正确；
2. 金额单位是否一致；
3. 百分比字段是否是小数形式，例如 5% 应为 `0.05`；
4. `unmapped_fields` 是否为空，若不为空，说明有字段未被别名表识别。

### 13.4 未识别字段处理

如果输入 CSV 有字段：

```text
未知字段
```

输出中会保留为：

```text
source_未知字段
```

同时 `unmapped_fields` 会记录：

```text
未知字段
```

这样不会丢失数据，也能提醒你把常用字段补进 `field_aliases.csv`。

### 13.5 当前限制

1. 字段映射只转换表头，不自动转换单位；
2. 不自动判断百分比是 `5` 还是 `0.05`；
3. 不自动判断金额单位是元、万元、百万美元还是亿元；
4. 外部数据不会自动进入每周脚本，建议人工检查后再放入 `data/raw` 参与筛选。

## 14. 数据质量检查层 V1

当前已经增加数据质量检查层，用于在评分前发现字段缺失、单位疑似错误、逻辑异常和行业分组风险。

| 文件 | 用途 |
|---|---|
| `data_quality_checks.py` | 数据质量检查主程序 |
| `scripts/run_data_quality_checks.ps1` | 手动执行质检 |
| `outputs/data_quality_*_issues.csv` | 问题明细 |
| `outputs/data_quality_*_report.md` | 中文质检报告 |

### 14.1 检查范围

当前规则包括：

| 类型 | 说明 |
|---|---|
| 必填字段缺失 | `ticker`、`company_name`、`industry` |
| 核心数值缺失 | `market_cap`、`net_income_ttm` |
| 百分比单位疑似错误 | 如毛利率填成 `45` 而不是 `0.45` |
| 行业未映射 | `industry_mapping_status = unmapped` |
| 行业样本不足 | `industry_median_sample_count < 5` |
| EV 逻辑异常 | 净债务为正但 EV 小于市值 |
| 资本开支符号疑似错误 | `capex` 为正数 |
| 风险字段缺失 | `audit_opinion`、`risk_flag` 为空 |

### 14.2 问题级别

| 级别 | 含义 |
|---|---|
| 严重 | 会明显影响识别公司或估值逻辑，例如缺少股票代码、EV 逻辑异常 |
| 警告 | 可能影响评分质量，例如单位疑似错误、行业未映射 |
| 提示 | 需要补充但不一定阻断评分，例如风险字段缺失、行业样本较少 |

### 14.3 手动执行

示例：

```powershell
powershell.exe -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_data_quality_checks.ps1 -InputPath F:\chatgptssd\project2\data\raw\sample_stocks.csv -IssuesPath outputs\data_quality_sample_issues.csv -ReportPath outputs\data_quality_sample_report.md
```

输出：

```text
outputs/data_quality_sample_issues.csv
outputs/data_quality_sample_report.md
```

### 14.4 每周脚本中的处理方式

每周脚本会在构建 `data/weekly_raw` 后，对其中每个 CSV 运行质检，并输出：

```text
outputs/data_quality_<文件名>_issues.csv
outputs/data_quality_<文件名>_report.md
```

当前规则会把“严重”问题接入评分阶段：存在严重问题的股票会被标记为 `data_quality_status = blocked`，总分强制为 0，等级为 `E`，不会进入候选池。

### 14.5 当前限制

1. 只做规则检查和严重问题阻断，不做自动修复；
2. 不自动识别金额单位，例如元、万元、百万美元；
3. 不自动识别百分比口径，只提示疑似问题；
4. 每周低估报告会展示质检摘要；完整明细仍以独立质检报告为主。

## 15. 严重问题阻断/降级规则 V1

当前已经把数据质量检查接入最终评分逻辑。

### 15.1 阻断规则

如果某只股票存在 `severity = 严重` 的数据质量问题，则：

| 字段 | 结果 |
|---|---|
| `data_quality_status` | `blocked` |
| `data_quality_block_reason` | 严重问题代码，例如 `missing_required_field` |
| `total_score` | `0` |
| `grade` | `E` |
| `action` | `数据质量阻断，暂不评分` |
| 候选池 | 不进入 |

当前会触发阻断的严重问题包括：

| 问题代码 | 说明 |
|---|---|
| `missing_required_field` | 缺少 `ticker`、`company_name` 或 `industry` |
| `enterprise_value_logic_error` | 净债务为正但 EV 小于市值 |

### 15.2 周报展示

每周报告会展示“数据质量摘要”，包括：

```text
问题总数
严重数量
警告数量
提示数量
问题代码分布
```

被阻断的股票会出现在每周报告的“高风险或剔除提示”中，备注会显示：

```text
数据质量阻断：...
```

这样高分股票不会因为数据错误误入候选池，同时也不会悄悄消失。

### 15.3 后续可扩展

后续可以把部分“警告”升级为阻断，例如：

1. 毛利率、ROE、股息率等百分比字段明显大于 1；
2. 行业未映射；
3. 行业中位数样本数太少；
4. 审计意见或风险标记缺失；
5. 市值、净利润、经营现金流等核心字段缺失。
