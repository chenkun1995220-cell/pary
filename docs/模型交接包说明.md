# 开发治理交接包说明

开发治理不绑定具体模型名称或版本。交接包用于记录当前执行环境、治理契约、验证证据和风险边界，不代表多模型自动调度。

提交前必须生成：

- `outputs/automation/latest_model_handoff_review.json`
- `outputs/automation/latest_model_handoff_review.md`

生成入口：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_model_handoff_review.ps1
```

该脚本只读取 `outputs/automation/latest_medium_term_goal_review.json` 及既有复核产物，不抓取行情、不重新评分、不修改正式模型参数。

交接包记录：

- 当前模块及完成度；
- `development_execution_profile`；
- `review_policy`；
- `environment_compatibility_policy`；
- `model_version_pinned=false`；
- `upgrade_compatibility_required=true`；
- 能力自适应执行摘要；
- 质量复核清单；
- 升级兼容契约；
- 验证命令和风险边界。

`run_pre_submit_review.ps1` 将交接包作为必需输入。以下情况不得放行：

- 交接包缺失、过期或结构版本不是 2；
- 仍包含遗留固定模型协作字段；
- 执行配置、复核策略或环境兼容策略不符合治理契约；
- 固定了模型版本；
- 未要求模型或环境升级后重新验证；
- 质量复核清单、完整测试命令或失败回退边界缺失；
- 正式模型修改边界不安全。
