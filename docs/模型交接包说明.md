# 模型交接包说明

当前项目仍未启用真正的自动双模型协作。`gpt5.3-codex-spark` 与 `gpt5.5` 目前是工程纪律分工口径，不是两个模型自动调度开发。

为了让该分工从文档口径落到可检查产物，每次进入提交前复核前必须生成：

- `outputs/automation/latest_model_handoff_review.json`
- `outputs/automation/latest_model_handoff_review.md`

生成入口：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File F:\chatgptssd\project2\scripts\run_model_handoff_review.ps1
```

该脚本只读取 `outputs/automation/latest_medium_term_goal_review.json`，不抓取行情，不重新评分，不修改正式模型参数。交接包记录当前模块、模块完成度、中期目标整体完成度、真实执行模式、验证命令、风险边界和 `gpt5.5` 口径复核清单。

`run_pre_submit_review.ps1` 会把交接包作为必需输入检查。若交接包缺失、声称已经启用自动双模型协作、执行模式不是 `single_codex_with_gpt55_review_checklist`，或缺少复核清单，提交前复核不得放行。
